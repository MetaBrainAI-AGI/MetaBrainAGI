## The problem

[VisionOpenSOC](VisionOpenSOC.md) secures one asset beautifully. But a real organization runs many repos across many hosts — and the questions get harder: *Does the running system still match what we audited? What's the posture across the whole fleet? How do we share hardened security skills across teams without leaking findings? How do we codify a response so it's consistent every time?*

## The solution

VisionOpenSOC-Pro is the commercial tier built on the open core. It adds continuous runtime attestation, cross-asset fleet posture, secure cross-team skill sharing, and gated auto-response playbooks — for a team and a fleet, not just one repo. All aggregation is over **signed posture summaries, never raw findings**, so nothing about your code leaves the box. It stays as local-first as the open core.

## What you get

- **Continuous runtime attestation.** Verify that the running system still matches what was audited — drift between "what we checked" and "what's actually live" is caught continuously.
- **Fleet posture.** Roll many repos and hosts into one view, with cross-asset CVE correlation, so a vulnerability that spans the fleet shows up as one signal, not twenty disconnected alerts.
- **Secure cross-team skill sharing.** Signed, license-gated skill bundles let teams share hardened security playbooks without exposing the underlying findings.
- **Auto-response playbooks.** Codified, gated runbooks — composing `vision-self-heal` — so a response is consistent and repeatable, with the dangerous steps held for approval.
- **Privacy by construction.** Aggregation is over signed posture *summaries*, never raw findings; nothing about your code leaves the box.

## Why trust it

- **Findings never leave the box** — fleet aggregation works on signed summaries only, never raw findings.
- **Built on an open, auditable core** — Pro extends the Apache-2.0 VisionOpenSOC you can already read and fork.
- **Gated by design** — auto-response playbooks hold the dangerous, irreversible steps for explicit approval.

## Who it's for

Security teams and platform owners running many repos and hosts who need fleet-wide posture, runtime assurance, and a consistent, codified response — without giving up local-first privacy.
